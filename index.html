Package panda

The panda python extension (pypanda) is a module built for interacting with the PANDA project. The project enables interaction and callbacks with a use-specified virtual machine.

It's likely most of what you want is either in panda.main or its listed ancestors.

Check out our example plugins here: https://github.com/panda-re/panda/tree/master/panda/python/examples

Sub-modules

panda.arm

Helper functions and information sepecific to the ARM architecture …

panda.asyncthread

Module to run a thread in parallel to QEMU's main cpu loop. Enables queuing up python functions from main thread and vice versa

panda.autogen

Automatically generated python code. Contains panda_datatypes, which is the glue of this project.

panda.blocking_mixins

Utilities to provide blocking interactions with PANDA. This includes serial and monitor interactions as well as file copy to the guest.

panda.callback_mixins

Mixin for handling callbacks and generation of decorators that allow users to register their own callbacks such as panda.cb_before_block_exec()

panda.decorators

The decorator submodule provide the basis for syntactic sugar on callbacks.

panda.extras

Extras are a bit like python-based plugins. You can import them, instantiate them as classes, and use their functionality.

panda.ffi_importer

This module is a workaround to maintain state of and not recreate cffi objects between mixins. All it does is import cffi and give you a handle to ffi.

panda.gdb_mixins

Provides the ability to interact with a QEMU attached gdb session by setting and clearing breakpoints. Experimental.

panda.hooking_mixins

Provides the ability to interact with the hooks2 plugin and receive callbacks based on user-provided criteria.

panda.images

Provides utilities for default images.

panda.libpanda_mixins

Methods that directly pass data to/from PANDA with no extra logic beyond argument reformatting.

panda.libqemu_mixins

Methods that directly pass data to/from QEMU with no extra logic beyond argument reformatting …

panda.main

The main module is a container for the Panda class.

panda.osi_mixins

Convenience functions to interact with the Operating System Instrospection (OSI) class of plugins.

panda.panda_expect

Custom library for interacting/expecting data via serial-like FDs

panda.plog

Functionality to interact with the PANDA Log.

panda.plugin_list

Wrapper class around list of active C plugins

panda.pyperiph_mixins

Mixin to allow interaction with pyperipherals.

panda.taint

Structures to support the taint subsystem.

panda.taint_mixins

Convenience methods for interacting with the taint subsystem.

panda.utils

Useful utilities for use in pypanda.

panda.volatility_cli_classes

Second method of interacting with volatility via their CLI options. Less preferred. Highly experimental.

panda.volatility_mixins

Utilities to integrate Volatility with PANDA. Highly experimental.

panda.x86

Helper functions and information sepecific to x86 and x86_64 …
